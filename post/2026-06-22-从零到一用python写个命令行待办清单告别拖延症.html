从零到一：用Python写个命令行待办清单，告别拖延症

📅 2026-06-22  ·  🏷️ 编程开发

为什么你需要一个自己的待办清单工具？

你是不是也用过各种待办清单App？从Todoist到TickTick，功能越来越花哨，但真正坚持用下来的有几个？我统计过身边30个朋友，超过80%的人在下载后两周内就放弃了。原因很简单：这些工具要么太复杂，要么不免费，要么数据不在自己手里。

今天，我就带你用Python从零写一个命令行待办清单。它轻量、免费、数据完全由你掌控，而且你还能随时给它加新功能。

开始前的准备

你需要Python 3.6+（建议3.8以上），以及一个文本编辑器。不需要任何第三方库——我们只用内置的json和datetime模块。

打开终端，新建一个文件叫todo.py，我们开始吧。

核心数据结构

待办事项最核心的信息是什么？任务描述、是否完成、优先级、创建时间。我们用字典表示每一项：

task = {
    "id": 1,
    "description": "写博客文章",
    "done": False,
    "priority": 3,  # 1最高，5最低
    "created_at": "2025-03-21 10:30:00"
}

整个清单就是一个列表：tasks = []。

数据持久化：用JSON文件保存

命令行程序一关，数据就丢了可不行。我们用JSON文件保存，简单又通用：

import json
import os

DATA_FILE = "tasks.json"

def load_tasks():
    if not os.path.exists(DATA_FILE):
        return []
    with open(DATA_FILE, "r") as f:
        return json.load(f)

def save_tasks(tasks):
    with open(DATA_FILE, "w") as f:
        json.dump(tasks, f, indent=2)

小提示：JSON文件是纯文本，你可以随时用记事本打开查看或修改。数据所有权完全在你。

添加任务：带优先级和日期

我们写一个add_task函数，支持优先级和自动生成时间：

from datetime import datetime

def add_task(description, priority=3):
    tasks = load_tasks()
    new_id = max([t["id"] for t in tasks], default=0) + 1
    task = {
        "id": new_id,
        "description": description,
        "done": False,
        "priority": priority,
        "created_at": datetime.now().strftime("%Y-%m-%d %H:%M:%S")
    }
    tasks.append(task)
    save_tasks(tasks)
    print(f"✅ 已添加：{description}（优先级：{priority}）")

列出任务：按优先级排序

光添加不行，还得能看。我们按优先级从高到低排列，并显示完成状态：

def list_tasks():
    tasks = load_tasks()
    if not tasks:
        print("🎉 没有待办事项，休息一下吧！")
        return
    # 按优先级排序，1最高
    sorted_tasks = sorted(tasks, key=lambda t: (t["done"], t["priority"]))
    print("\n📋 待办清单：")
    print("=" * 40)
    for t in sorted_tasks:
        status = "✅" if t["done"] else "⬜"
        prio = "⭐" * (6 - t["priority"])  # 1优先级显示5颗星
        print(f"{status} [ID:{t['id']}] {t['description']} {prio}")
        print(f"   创建于：{t['created_at']}")
    print("=" * 40 + "\n")

完成任务和删除任务

def done_task(task_id):
    tasks = load_tasks()
    for t in tasks:
        if t["id"] == task_id:
            t["done"] = True
            save_tasks(tasks)
            print(f"🎯 任务 {task_id} 已完成！")
            return
    print("❌ 未找到该任务")

def delete_task(task_id):
    tasks = load_tasks()
    original_len = len(tasks)
    tasks = [t for t in tasks if t["id"] != task_id]
    if len(tasks) < original_len:
        save_tasks(tasks)
        print(f"🗑️ 已删除任务 {task_id}")
    else:
        print("❌ 未找到该任务")

命令行入口

我们用sys.argv接收命令行参数，这样你就可以在终端直接运行：

import sys

def main():
    if len(sys.argv) < 2:
        print("使用方法：python todo.py <命令> [参数]")
        print("命令：add <描述> [优先级]，list，done <id>，delete <id>")
        return

    command = sys.argv[1]

    if command == "add":
        if len(sys.argv) < 3:
            print("请提供任务描述")
            return
        description = sys.argv[2]
        priority = int(sys.argv[3]) if len(sys.argv) > 3 else 3
        add_task(description, priority)
    elif command == "list":
        list_tasks()
    elif command == "done":
        if len(sys.argv) < 3:
            print("请提供任务ID")
            return
        done_task(int(sys.argv[2]))
    elif command == "delete":
        if len(sys.argv) < 3:
            print("请提供任务ID")
            return
        delete_task(int(sys.argv[2]))
    else:
        print(f"未知命令：{command}")

if __name__ == "__main__":
    main()

实际使用示例

打开终端，在你的项目文件夹里运行：

# 添加任务
python todo.py add "完成Python项目文档" 2
# 添加另一个
python todo.py add "买牛奶" 5
# 查看清单
python todo.py list
# 完成任务ID为1的任务
python todo.py done 1
# 删除任务ID为2的任务
python todo.py delete 2

进阶功能：统计与优化

想让工具更实用？加个统计功能：

def stats():
    tasks = load_tasks()
    total = len(tasks)
    done = sum(1 for t in tasks if t["done"])
    pending = total - done
    completion = (done / total * 100) if total > 0 else 0
    print(f"📊 统计：总计{total}项，已完成{done}项，待办{pending}项")
    print(f"完成率：{completion:.1f}%")
    if pending > 0:
        print(f"💡 建议：先做优先级最高的任务！")

另外，当任务数量超过1000条时，JSON读写会变慢。这时可以考虑改用SQLite，或者按日期分文件存储。对于个人使用，1000条以内JSON完全够用。
